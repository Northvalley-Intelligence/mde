# Context Packs

A context pack is reusable MDE knowledge that applies to a class of future work.

Context packs should be loaded just-in-time when a generation matches relevant artifact profiles. They are not instructions to modify every repository immediately.

Current context packs:

- `client-success-crm.md`: client success CRM and follow-up workflows.
- `client-website-build.md`: client website repo, technical stack, Cloudflare deployment, GitHub governance, and client-safe MDE rules.
- `mobile-companion-app.md`: mobile companion app planning and validation.
- `pdf-generation.md`: PDF, print layout, client-facing document, and report generation work.
- `personalized-family-software.md`: personalized family software workflows and role/privacy constraints.
- `ui-forms.md`: form workflows, web UI, and mobile UI work.
- `printable-artifacts.md`: official printable and business document work.
- `website-assessment-to-build.md`: turns website assessment evidence into SEO/AEO, service-area, content, validation, and Codex build guidance.

Use `knowledge/issue-signatures.jsonl` to connect known issue signatures to the relevant context pack and validator candidate.
