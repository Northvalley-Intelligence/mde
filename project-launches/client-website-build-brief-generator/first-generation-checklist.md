# First Generation Checklist

- Create local MDE files.
- Confirm upstream assessment data sources.
- Define client intake schema ideas.
- Define stale assessment checks.
- Define missing client detail checks.
- Define confidentiality checks.
- Define launch pack output contract: `codex-build.prompt.md`, mission, intake summary, content-needed checklist, technical brief, SEO/AEO strategy, validation plan, and deployment checklist.
- Define assessment evidence traceability checks.
- Define SEO/AEO service-area coverage checks.
- Define Northvalley GitHub organization and repo naming checks.
- Define medinaclean-style technical baseline checks.
- Define Cloudflare Workers/OpenNext deployment checks.
- Define `main`/`prod` branch synchronization checks for projects that use `prod`.
- Generate first sample launch pack from controlled sample assessment data.
- Validate the sample launch pack against `validation-strategy.seed.json`.
- Record outbox event and learning triage.
