# Validators

This directory holds reusable validator ideas and shared validation notes.

Validator candidates should begin in `ledger/lessons.jsonl` or `ledger/skill-update-candidates.jsonl` before becoming shared tooling. Keep validators project-specific until repeated evidence shows they are broadly useful.

Initial candidates:

- Schema compatibility validation for assessment data contracts.
- Proposal and report formatting validation.
- Print-mode validation for vouchers, invoices, flyers, and other business artifacts.
- Usability validation for preserving user input after validation failure.

Documented validator candidates:

- `validators/pdf/pdf-margin-safety.md`
- `validators/usability/form-data-preservation.md`
- `validators/ui-standards/affordance-integrity.md`
