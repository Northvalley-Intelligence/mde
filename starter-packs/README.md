# Starter Packs

Starter packs are MDE startup guidance, not static code templates.

Use them with the current MDE skill and central context. They should help a new repo begin with mission clarity, project-specific validation strategy, local MDE structure, and safe handoff behavior.

Current starter packs:

- `client-website-build-brief-generator`
- `client-success-crm`
- `mobile-companion-app`
- `new-client-website`
