# Promote Project Learning

Every meaningful generation should end with MDE Learning Triage.

Ask:

1. What did we learn?
2. Is it project-only learning?
3. Is it a reusable MDE pattern?
4. Is it a validator improvement?
5. Is it a skill update candidate?
6. Is it a content, blog, or book insight?
7. Is it a business process improvement?

## Promotion Rules

Do not promote everything.

Promote only if the lesson is likely to recur across projects, improves validation or generation quality, reduces future maintenance cost, improves personalized software delivery, or helps explain MDE publicly.

## Examples

Jacob Family UI bug:

- Project fix: preserve form data after validation failure.
- MDE lesson: add usability validator for data preservation.
- Content seed: AI-generated apps can pass BDDs but fail recovery usability.

Voucher formatting bug:

- Project fix: update print CSS or template.
- MDE lesson: printable artifacts require print-mode validation.
- Validator candidate: PDF or print layout validation.
- Content seed: printed business artifacts need validation, not just visual review.
