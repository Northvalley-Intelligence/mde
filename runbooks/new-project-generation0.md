# New Project Generation 0

Generation 0 is not implementation.

It creates mission clarity, project identity, validation strategy, applicable context, initial task graph, risk register, and readiness assessment.

Required outputs:

- `MISSION.md`
- `MISSION_UPDATES.md`
- `.mde/project.json`
- `.mde/state.json`
- `.mde/validation-strategy.json`
- `.mde/outbox/events.jsonl`
- `.mde/lessons-local.jsonl`
- initial BDD inventory
- initial task graph
- initial risk register
- Generation 0 readiness assessment

Validation categories are reusable. Validation tools are project-specific.

Phase exit uses two independent Validation Gate passes.
