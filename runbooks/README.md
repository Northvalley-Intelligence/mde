# Runbooks

Runbooks describe repeatable cross-repo MDE work.

Use them when a change affects another repository, a report or proposal must be regenerated, a project lesson may become reusable, repository ownership or GitHub submission needs to be checked or rolled out, or an existing Codex session needs to be bridged.

Start with `session-routing.md` when deciding which prompt to use. Use `just-in-time-learning.md` before implementation in MDE-aware projects. Use `generation0-validation-strategy.md` before implementation in brand-new projects or first safe validation updates.
