# New Client Website Bootstrap

Use this when starting a brand-new client website repo or folder.

1. Create or open a client-specific repo.
2. Use `prompts/new-client-website-skill-bootstrap.prompt.md`.
3. Load current central MDE context and relevant assessment output.
4. Run Generation 0 before implementation.
5. Create client-safe MDE artifacts only.
6. Generate client questions and content-needed checklist.
7. Generate website build mission and technical brief.
8. Generate launch validation plan.
9. Record local outbox event.
10. Run session-end learning triage.

Do not copy old client website code blindly.
Do not place central private ledgers in a client-owned candidate repo.
